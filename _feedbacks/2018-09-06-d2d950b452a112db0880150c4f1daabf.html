Hi, thanks for all your blogs, they do teach me a lot, in my current project, I adopt your way of handling errors, and it works for me most of the time. https://graphqlme.com/2018/03/17/i-thought-of-a-better-way-to-handle-errors/.

But I think it does not cope with validation error (for example, json web token expired) so well, becuase:

1. expiration should be handled out of business logic in client, namely, if you go expired, login again, don't handle the business anymore
2. it is stupid to issue query like:

{
getA(...) {...}
getB(...) {...}
}

and get results like:

{
data: {
getA {
error: { code: 'TOKEN_EXPIRED' }
},
getB {
error: { code: 'TOKEN_EXPIRED' }
}
}
}

so I suggest that this particular kind of error should be put at the top level, like:

{
error: {
errors: {
[code: 'TOKEN_EXPIRED']
}
}
}

but this way, it will mess with the type system error, as in your blog "https://graphqlme.com/2017/12/31/stop-fighting-the-type-system/". I wish you could give me a better way to handle this kind of error, thanks very much!
